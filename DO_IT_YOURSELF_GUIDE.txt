-----------------------

PART 2: SET UP SUPABASE DATABASE
========================================

Step 1: Create Supabase Project
--------------------------------
1. Go to: https://supabase.com/dashboard
2. Click "New Project"
3. Fill in:
   - Name: EventKaro Production
   - Database Password: (choose strong password - SAVE THIS!)
   - Region: Mumbai or Singapore
4. Click "Create new project"
5. WAIT 2-3 minutes for project setup


Step 2: Run Database Migrations
--------------------------------
In Supabase Dashboard:

1. Click "SQL Editor" in left sidebar
2. Click "+ New Query"
3. Run migrations IN THIS ORDER (3 total):

MIGRATION 1 - Initial Schema:
-------------------------------
File: C:\Users\Bhavesh\Desktop\in\EventKaro\supabase\migrations\20241209000000_initial_schema.sql

In Supabase SQL Editor:
1. Click "+ New Query"
2. Open file: 20241209000000_initial_schema.sql
3. Copy ALL 356 lines
4. Paste into Supabase SQL Editor
5. Click "Run" (bottom right)
6. Wait for "Success" message

Creates: Organizations, Events, Tickets, Orders, Payments, Check-ins, Profiles


MIGRATION 2 - Guest Management + Accommodations:
------------------------------------------------

1. Click "+ New Query" again
2. Open file: 20241209010000_guest_management.sql
3. Copy ALL 230 lines
4. Paste into Supabase SQL Editor
5. Click "Run"
6. Wait for "Success"

MIGRATION 3 - Vendor Marketplace:

Step 3: Get API Keys
--------------------
In Supabase Dashboard:

1. Click "Settings" (gear icon bottom left)
2. Click "API" in settings menu
3. COPY these three values (you'll need them for Vercel):

   PROJECT URL:
   (looks like: https://xxxxxxxxxxxxx.supabase.co)
   
   ANON PUBLIC KEY:
   (long string starting with "eyJ...")
   
   SERVICE ROLE KEY:
   (long string starting with "eyJ..." - KEEP THIS SECRET!)

SAVE these keys in a text file!

PART 3: DEPLOY TO VERCEL
========================================

Step 1: Import Project to Vercel
---------------------------------
1. Go to: https://vercel.com/new
2. Sign in with GitHub
3. Click "Import Git Repository"
4. Select your "EventKaro" repository
5. Click "Import"


Step 2: Add Environment Variables
----------------------------------
In the "Configure Project" screen, add these:

Click "Environment Variables" section

Variable 1:
Name:  NEXT_PUBLIC_SUPABASE_URL
Value: (paste your Supabase Project URL from Step 3 above)

Variable 2:
Name:  NEXT_PUBLIC_SUPABASE_ANON_KEY
Value: (paste your Supabase Anon Public Key)

Variable 3:
Name:  SUPABASE_SERVICE_ROLE_KEY
Value: (paste your Supabase Service Role Key)


Step 3: Deploy
--------------
1. Click "Deploy"
2. WAIT 2-3 minutes
3. You'll see "Congratulations!" when done
4. Copy your deployment URL (looks like: https://eventkaro-xxxxx.vercel.app)


Step 4: Update Supabase Auth
-----------------------------
Go back to Supabase:

1. Authentication â†’ URL Configuration
2. Add your Vercel URL to "Redirect URLs":

   https://your-actual-vercel-url.vercel.app/auth/callback

3. Click "Save"

TROUBLESHOOTING
========================================

Problem: Git push fails with "Permission denied"
Solution: Use Personal Access Token instead of password
         https://github.com/settings/tokens

Problem: Supabase migration fails
Solution: Make sure you ran migrations in correct order
         Run them one by one, wait for "Success" each time

Problem: Vercel build fails
Solution: Check environment variables are correct
         No extra spaces in keys/values

Problem: Auth not working on deployed site
Solution: Make sure you added Vercel URL to Supabase redirect URLs
         Format: https://your-app.vercel.app/auth/callback
